Music Streaming Interface Project Documentation

1. Project Overview & Objectives

Problem Statement

Users often face slow, ad-heavy, or complex interfaces when streaming music online. This project aims to build a lightweight, responsive, and ad-free music streaming interface that focuses on usability, speed, and personalized playlists.

Objectives

  • Provide seamless access to an online music catalog.
  • Enable users to search, play, and manage playlists easily.
  • Design a clean and intuitive UI/UX with smooth transitions and audio controls.
  • Allow admins to manage songs and metadata efficiently.

Key Features

  • User authentication (Register/Login)
  • Browse and search songs by name, artist, or genre
  • Integrated audio player (Play, Pause, Skip)
  • Playlist creation and management
  • Admin dashboard to manage songs
  • Responsive and mobile-friendly UI

Expected Outcome

A functional web app prototype where users can play and organize songs seamlessly, admins can maintain the music catalog, and the interface works efficiently on both desktop and mobile.

2. Technology Stack & Environment Setup

Frontend

  • Framework: React.js (Vite setup for fast builds)
  • Styling: Tailwind CSS for responsive design
  • State Management: Context API or Redux Toolkit
  • Audio Playback: HTML5 Audio API

Backend

  • Runtime: Node.js
  • Framework: Express.js
  • Middleware: CORS, JSON parser, JWT authentication
  • File Handling: Multer (for audio uploads, optional)

Database

  • MongoDB (hosted on MongoDB Atlas)
  • ODM: Mongoose

Development Tools

  • VS Code – Code editor
  • Postman – API testing
  • GitHub – Version control & repo management
  • Render / Vercel – Hosting
  • Nodemon – Hot reload during backend dev

Environment Setup

3. API Design & Data Model

Planned REST Endpoints

MethodEndpointDescriptionAccess
POST/api/auth/registerRegister a new userPublic
POST/api/auth/loginUser login and token generationPublic
GET/api/songsFetch all songsUser
GET/api/songs/:idGet specific song detailsUser
POST/api/playlistsCreate a new playlistUser
GET/api/playlists/:userIdGet all playlists of a userUser
DELETE/api/playlists/:idDelete playlistUser
POST/api/admin/songAdd new songAdmin
PUT/api/admin/song/:idEdit song infoAdmin
DELETE/api/admin/song/:idDelete songAdmin

Database Schema


// User Schema
{
  username: String,
  email: String,
  password: String,
  playlists: [{ type: mongoose.Schema.Types.ObjectId, ref: "Playlist" }]
}

// Song Schema
{
  title: String,
  artist: String,
  genre: String,
  url: String,
  coverImage: String

// Playlist Schema
{
  name: String,
  userId: { type: mongoose.Schema.Types.ObjectId, ref: "User" },
  songs: [{ type: mongoose.Schema.Types.ObjectId, ref: "Song" }]
}

4. Front-End UI/UX Plan

Wireframes & Pages

  • Login / Register: Authentication screen
  • Home Page: Trending songs & search bar
  • Song List: Shows results with play buttons
  • Player Bar: Sticky bottom bar for Play/Pause/Skip
  • Playlist Page: View, create, and edit playlists
  • Admin Dashboard: Manage songs and uploads

Navigation Flow


Login → Home → Song List → Player → Playlist → Logout
                 ↳ Admin → Add/Edit/Delete Songs

State Management Approach

  • Global State: Context API (Auth, Playlist, Player state)
  • Local State: useState for forms and UI toggles
  • API Calls: Axios hooks
  • Persistent Data: JWT stored in localStorage
